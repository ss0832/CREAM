# Rust API

This page is a short map of where each public Rust type lives. The
authoritative per-symbol documentation is generated by `rustdoc`.

For the Python package (`cream-python`) see
[Python API](python_api.md) instead.

## Engines

### `cream::ComputeEngine`

The main GPU-backed engine. Owns a `wgpu::Device` and `wgpu::Queue`.
Constructed asynchronously with
`ComputeEngine::new(NeighborStrategy) -> impl Future`.

Key methods:

- `compute(&mut self, positions, atom_types, cell, potential) -> impl Future<Output = Result<ComputeResult, CreamError>>`
- `compute_sync(...)` ‚Äî blocking wrapper around `compute`
- `compute_with_debug(...)` ‚Äî returns a `ComputeResult` plus a
  `CellListDebugReadback` with per-phase wall-clock timings and shader
  counters. Used by the diagnostic binaries.

### `cream::CpuEngine`

Pure-Rust CPU engine, rayon-parallelised. Constructed with
`CpuEngine::new()` ‚Äî always succeeds, no GPU required.

Key methods:

- `compute_sync(&self, positions, atom_types, cell, potential) -> Result<ComputeResult, CreamError>` ‚Äî O(N¬≤) half-pair
- `compute_cell_list_sync(&self, positions, atom_types, cell, potential) -> Result<ComputeResult, CreamError>` ‚Äî O(N) Morton cell list

### `cream::ComputeResult`

```rust
pub struct ComputeResult {
    pub forces: Vec<[f32; 3]>,        // eV/√Ö, length N
    pub energy: f32,                   // total energy, eV
    pub energy_per_atom: Vec<f32>,     // length N, sum ‚âà energy
}
```

## Potentials

### `cream::potential::eam::EamPotential`

LAMMPS / NIST `.eam.alloy` parser. Supports single-element and
multi-element alloy files.

Key constructors:

- `EamPotential::from_file(&Path) -> Result<Self, CreamError>`
- `EamPotential::from_str(&str) -> Result<Self, CreamError>`

### `cream::potential::NeighborStrategy`

```rust
pub enum NeighborStrategy {
    AllPairs,
    CellList { cell_size: f32 },
}
```

Selects the neighbour-list algorithm. See [Backends](backends.md).

### `cream::potential::GpuPotential`

Trait every physics model must implement. Decouples the force engine
from the potential functional form. See [Architecture](architecture.md#layered-structure).

## Errors

### `cream::CreamError`

Single error enum covering every failure mode. Uses `thiserror` for
`Display` and `Error` impls.

Variants:

- `DeviceLost(String)` ‚Äî GPU device lost or no suitable adapter.
- `OutOfMemory { requested_mb }` ‚Äî requested GPU allocation too large.
- `InvalidInput(String)` ‚Äî mismatched lengths, bad cell, etc.
- `ParseError { line, message }` ‚Äî `.eam.alloy` parse failure.
- `ShaderError(String)` ‚Äî WGSL compilation or composition failure.
- `CellListError(String)` ‚Äî cell-list construction failure.
- `BufferMapError(String)` ‚Äî GPU buffer readback failure.

## Helpers

- `cream::ortho_cell(lx, ly, lz) -> [[f64; 3]; 3]` ‚Äî orthorhombic cell
  constructor for the `reference.rs` f64 reference.
